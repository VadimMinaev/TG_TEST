# План упорядочивания дизайна

## Анализ текущего состояния

**Что есть:**
- Tailwind CSS v4.1.3 + **1818 строк** кастомного CSS в `globals.css`
- `src/components/ui/` — **45 компонентов shadcn/ui** (Radix UI) — большинство **не используются**
- Свои CSS-классы: `.panel`, `.card`, `.split-layout`, `.entity-view`, `.table-basic`, `.icon-button` и т.д.
- Каждая страница стилизует формы по-своему (инлайн Tailwind, свои классы, `.users-*` классы)
- Тёмная тема с градиентами, blur, glassmorphism — выглядит неплохо, но **шумно**
- Login page — отдельный дизайн с орбами и анимациями (130+ строк CSS)

**Главные проблемы:**
1. **Два стиля компонентов**: shadcn/ui есть, но страницы используют ручные CSS-классы
2. **1818 строк кастомного CSS** — много дублирования и мёртвого кода
3. **Неконсистентность**: каждый page формирует формы и таблицы по-своему
4. **Визуальный шум**: чрезмерные градиенты, blur, тени на каждом элементе
5. **Мёртвый код**: классы `.users-*`, `.dashboard-*` дублируют общие паттерны

## Предлагаемые шаги (по приоритету)

### 1. Привести формы к shadcn/ui компонентам
**Файл**: каждая страница с формами (`AiBots.tsx`, `Bots.tsx`, `Rules.tsx`, `Integrations.tsx`, `Users.tsx`)

Сейчас формы используют инлайн Tailwind:
```tsx
<input className="w-full rounded border border-[hsl(var(--input))] bg-[hsl(var(--background))] px-3 py-2 font-mono" />
```

Заменить на shadcn/ui:
```tsx
import { Input } from '../components/ui/input';
import { Label } from '../components/ui/label';
<Label>Название</Label><Input placeholder="AI Ассистент" />
```

**Эффект**: -200 строк инлайн-стилей, единый вид полей ввода.

### 2. Заменить таблицы на shadcn/ui Table
**Файлы**: `AiBots.tsx`, `Bots.tsx`, `Integrations.tsx`, `Polling.tsx`

Сейчас:
```tsx
<table className="table-basic w-full border-collapse text-sm">
```

Заменить на:
```tsx
import { Table, TableBody, TableCell, TableHead, TableHeader, TableRow } from '../components/ui/table';
```

### 3. Упростить кастомный CSS — удалить мёртвый код
**Файл**: `src/styles/globals.css`

Удалить/Ḏorифицировать:
- `.users-*` классы (60+ строк) — не используются нигде
- Дубли `.dashboard-event-item` (определены дважды)
- Дубли `.dashboard-event-title-row` (определены дважды)
- Избыточные media queries

**Ожидаемый результат**: -150-200 строк CSS.

### 4. Унифицировать паттерн split-layout → Resizable Panels
**Файлы**: все страницы с split-layout

Сейчас `.split-layout` — фиксированный 420px слева. Есть `react-resizable-panels` в зависимостях, но не используется.

Вариант: перейти на `<ResizablePanelGroup>` для drag-resize боковой панели.

### 5. Убрать визуальный шум
**Файл**: `src/styles/globals.css`

Упростить:
- Фон `body` — убрать 2 радиальных градиента (или оставить один мягкий)
- `.split-layout` — убрать радиальные градиенты, оставить чистый фон
- `.card` — уменьшить `box-shadow` с 32px до 16px
- `.topbar` — убрать `backdrop-filter: blur(12px)` (не виден на практике)

### 6. Единый компонент Button
**Файлы**: все страницы

Сейчас используются: инлайн Tailwind, `.icon-button`, `.users-btn`, `.login-submit`

Заменить на shadcn/ui `Button` с variants:
```tsx
import { Button } from '../components/ui/button';
<Button variant="default">Сохранить</Button>
<Button variant="destructive">Удалить</Button>
<Button variant="secondary">Отмена</Button>
<Button variant="ghost" size="icon"><Pencil /></Button>
```
