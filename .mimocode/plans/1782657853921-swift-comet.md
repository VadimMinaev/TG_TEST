# Plan: Переделать RulesList из таблицы в карточки

## Проблема

RulesList использует `<table>` с 3 колонками (Название, Условие, Статус) в узком сайдбаре 264px. Текст условий ломается хаотично, строки слиплись, выглядит дешево.

Все остальные страницы (Bots, Polling, Integrations, Users, Accounts) используют паттерн `fp-bot-item` — карточки с аватаром, именем, подзаголовком и dot-индикатором. RulesList — единственная страница с таблицей.

## Решение

Переписать `src/components/RulesList.tsx` чтобы использовать паттерн `fp-bot-item` (как в FormPage.tsx):

- Аватар: первая буква имени
- Имя: rule.name (truncated)
- Подзаголовок: condition (truncated, моноширинный)
- Dot: вкл/выкл

Убрать `<table>`, заменить на `<div>` карточки.

## Файлы

- `src/components/RulesList.tsx` — переписать
- `src/styles/globals.css` — possibly tweak `.fp-bot-model` для更好的 truncation

## Верификация

- `npm run build`
- Визуально: сайдбар webhook должен выглядеть как сайдбар ботов/пуллингов
